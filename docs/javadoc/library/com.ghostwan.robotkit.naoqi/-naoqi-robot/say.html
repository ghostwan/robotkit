say

suspend fun say(@StringRes phraseRes: Int, @RawRes vararg animationsRes: Int, bodyLanguageOption: BodyLanguageOption = BodyLanguageOption.NEUTRAL, locale: Locale? = null, throwOnStop: Boolean = true, onStart: () -> Unit = null, onResult: (Result<Void>) -> Unit = null): Unit

Make the robot say a phrase using a string resource and optionally play an animation at the same time

If onResult it's not set, the coroutine will be suspended until the robot finish its speech without blocking the thread

Parameters

phraseRes - The phrase to say. It is an Android string resource.

animationsRes - Animations which will be played during the speech. They are android raw resources.

bodyLanguageOption - Body language policy

locale - The locale use to say something, if it's null use the one of the device.

throwOnStop - By default it's true. Does this method throw when the stop api is called and stop it. Could be useful to set it to false when we want to continue actions scenario after a stop

onStart - This lambada is called when the action start on the robot

onResult - If this lambada is set the coroutine won't wait and the result of the api call will be forward to the lambda. Give a Success or a Failure with the exception
